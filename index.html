HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Div tag is used as a container for HTML elements it can be styled using an id or class attribute. Any sort of content can be put inside a div tag
  • Span is an inline container used to mark up a part of a text, or a part of a document. It is much like a div but div is a block leve element and span is an inline element
  • The "section" element allows us to elegantly organize the content on our page based on the topics we have
  • The "id" attribute is used as a way to identify elements and select them

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • To external link the stylesheet place "link rel="stylesheet" href="./assets/style.css" under the last meta tag
  • To find hex color codes follow this link HTML Color Codes
  • Box model help MDN Box Model article
  • The box-shadow property adds shadow effects around an elements frame

Git

  • git status: checks what branch we are currently on
  • git checkout: allows you to named branch switch branches
  • git checkout -b branch-name: creates a new branch and switches to it
  • The order in which you should use git version control to save commit edits is as follows:
    1. git status - to check the branch and status
    2. git add -A : to commit all files in the working branch
    3. git commit -m "concise blurb on what you did entered here": this a a commit message that should be descriptive of the changes implemented
    4. git pull origin main : pulls the base branch into your feature branch before you push any code. (should tell you its already up to date *in most cases*)
    5. git push origin feature/"blahblahblah" : *note: no parentheses are needed* this pushes all changes to repo for pull request

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An "Array" is a single variable that is used to hold a group of data ex. var deskItems = ['mouse', 'keyboard', 'screen', laptop]
  • A "for" loop uses a predictable pattern to perform a task on all the items in an array using a single block of code that is executed over and over again. ex for(var x=0; x *greaterthan symbol* shapes.length; x++){block of code goes here}
  • conditional statements = if(something){than do something}
  • A Function is a set of instructions that tells the computer how to perform a certain task. A functions looks like this: function whateverfucntionName(){code block entered here.. conditional statements or for loops etc.} don't forget to call the function when you need it to work!!
  • Console.log regularly to check to ensure code is functioning properly in devtools
